Value of Data

Data is an essential ingredient for driving innovation and differentiation, and is the key to unlocking value from artificial intelligence. It powers AI-driven business insights, helps in real-time decision making and is basis of how organizations build their apps.

Organizations are not only focusing on sales, inventory or personnel data, but are also keen in extracting value from hidden data points from a myriad of sources and systems like how customers engage with you across social media platforms.

Capturing and storing such data can be complex.

Organizations that want to adapt must determine how to close the gap and support value generation. An intelligent data cloud is key to unlocking more business value.

Data Model Categorizations

  • Structured: Highly organized and well defined. E.g.: Data in CRM's, BPM's etc.
  • Semi-structured: Organized into a hierarchy but without differentiation or any particular ordering. E.g.: HTML, JSON, XML, emails etc.
  • Unstructured: Either don't have predefined data model or isn't organized in a predefined manner. E.g.: videos, social media posts, data files (documents, presentations), images, audio files etc.

With right tools businesses can extract value from unstructured data by using machine learning. Google's Cloud Vision API can help in identifying and even label the pictures to describe the contents.

Data Warehouses

A place to store data, unlike databases which are used to store, retrieve and use data, data warehouses are used for analysis and reporting of structured and semi-structured data from multiple sources. Data present anywhere in the organization can be stored here for analysis and reporting (like a central hub for all business data) - reporting can be ad hoc or custom.

Google's data warehousing solution is BigQuery

Data Lakes

A repository to ingest, store, explore, process and analyze any type of volumes of raw data, regardless of source. It can store different types of data in its original format; ignoring size limits, and without much pre-processing or adding structure.

This differs from a data warehouse that contains structured data that has been cleaned and processed, ready for strategic analysis based on predefined business needs.

Data lakes consists of several products depending on the type of data being ingested:

  • For structured data: Cloud SQL, Spanner or BigQuery
  • For semi-structured options: Bigtable and Datastore
  • For unstructured data: Cloud Storage

Key Difference: Data warehouse users are business intelligence analysts who are closer to business, data lake users are data scientists and data engineers who are close to raw data.

Data Categorizations

  • First party data: Proprietary customer datasets that a business collects from customer or audience transactions and interactions.
  • Second party data: First-party data from another organization, such as a partner or other business in their supply chain, that can be easily deployed to augment a company's internal datasets. Organizations don't directly own this data but is relevant to their business.
  • Third party data: Datasets collected and managed by organizations that don't directly interact with an organization's customers or business. These datasets might come from government, nonprofit, or academic sources, like weather or public demographic data, or from industry-specific sources like analyst reports or industry benchmarking.

Google's marketplace is a platform to purchase third party data.

No matter where you are in your company, you too can use data to solve challenges.

Data Value Chain

  • Data genesis: Initial creation of data, can be a click, swipe of card or a sensor record from IoT device
  • Data collection: Bringing initially created data to assembly line through ingestions (extracting data from where it is hosted and bringing it to a new system). Requirements can vary based on volume, speed and type of data that's required for a given analysis and how fast data needs to be analyzed.
  • Data processing: Where raw data is transformed into a form from which users can derive insights. It can be a simple single stage operation or a set of complex cascading steps.
  • Data storage: Where data lands and can be found and is ready for analysis and action. Different types of data can be stored in different ways e.g. NoSQL for faster R/W operations, data warehouses for faster access to analysis, object storage for unstructured data
  • Data analysis: Provides direction for business orientation
  • Data activation: Data is pushed to relevant business procedures and decision makers so that action can be taken. Common points of activation are apps that make automated decisions and business intelligence dashboards that guide humans towards better and more informed decisions.

Data Governance

An effective data governance program can help organizations democratize and embed data in every decision, while also ensuring data is secure and protected from unauthorized access.

Definition: Setting internal standards - internal policies - that apply to how data is gathered, stored, processed and disposed off. It governs who can access certain data and what data is under governance. It also involves complying with external standards set by industry associations, government agencies, and other stakeholders.

Data Governance Benefits

  • Make data more valuable
  • Users make better and more timely decisions
  • Improves cost controls
  • Enhances regulatory compliance
  • Earn trust from both suppliers and customers
  • Manage risk
  • Allows more personnel access to more data

Organizations Without Proper Data Governance

  • Suffer from compliance violations which can lead to fines, poor data quality
  • Delayed analysis and missed business opportunities
  • Poorly trained AI models which reduce models accuracy and benefits of using AI

Google Cloud Data Management Solutions

Unstructured Data Storage: Cloud Storage

Offers developers and IT organizations durable and highly available object storage. Object storage is a computer storage architecture which manages data as "objects" instead of files. Objects are stored in packaged form that contains the binary form of actual data and corresponding metadata - such as creation date, author, resource type, permissions and GUID.

Commonly stored objects include videos, pictures, audio recordings i.e. prominently used to store unstructured data.

Storage Classes

Standard

Best for frequently accessed data or data that is stored for brief moment

Nearline

Best for data accessed once a month or less

Coldline

Low cost option for infrequently accessed data compared to Nearline, suitable for data meant for reading or modifying once every 90 days

Archive

Lowest cost option used ideally for data archiving, data backup and disaster recovery, best for data that you plan to access less than once a year

Common Features

  • Unlimited storage with no minimum object size requirement
  • Worldwide accessibility and location
  • Low latency and high durability

Cloud Storage also provides Auto class which automatically transitions objects to appropriate class based on usage patterns.

Structured Data Storage

Cloud SQL

Offers fully managed relational databases, including MySQL, PostgreSQL, SQL Server. Google takes care of patching, updates, managing backups & configuring replications. It encrypts data when on Google's internal network and when stored in DB tables, temp files and backups. Includes network firewall, which controls access to each individual DB instance. Has availability greater than 99.95%.

Spanner

Fully managed, mission critical RDBMS, that scales horizontally to handle unexpected business spikes. Spanner is a service that powers Google's multi-billion dollar business. Suited for apps that require SQL relational database management system with joins & secondary indexes, built in high availability. Provides data redundancy to reduce downtime when a zone or instance becomes unavailable, strong global consistency, which ensures that all locations where data is stored are updated to most recent version. Has availability of up to 99.99%. Is ACID compliant cloud DB that automatically handles replicas, sharding, transaction processing.

Decision Guide: To decide among Cloud SQL and Spanner, if you have outgrown a RDBMS and are sharding your DBs for throughput high performance, need transactional consistency, global data, strong consistency, or just consolidate your DB consider Spanner. If you don't need horizontal scaling or globally available systems Cloud SQL is cost effective solution.

BigQuery

A fully managed data warehouse. Used to gather data from wide range of sources within an organization and is used to guide management decisions. Since it is fully managed BigQuery takes care of underlying infrastructure. Users can focus on using SQL queries to get answers to complex business questions, without having to worry about deployment, scalability & security.

Key Features:

  • Provides both storage and analytics in one place
  • Can store petabytes of data
  • Data is encrypted at rest by default without any actions from user
  • Seamless integrations with existing partner ecosystem
  • Works in multi-cloud environment
  • Built-in ML features - ML models can be written directly in BigQuery in SQL
  • Models can be seamlessly exported to Vertex AI

Semi-Structured Data Storage

Firestore

A flexible, horizontally scalable, NoSQL cloud DB for storing and syncing data in real-time. Can be accessed directly via mobile or web app. Firestore performs data storage in forms of documents, with documents being stored in collections. Documents support wide variety of data types such as nested objects, strings, numbers.

Main feature: Auto scaling. It scales automatically depending on user demands while maintaining the same performance irrespective of DB size. Also supports offline usage through comprehensive database in users devices.

Bigtable

Google's NoSQL big data DB service. Powers many core Google services including search, analytics, maps and Gmail. Designed to handle large workloads with low latency (responds to requests quickly) and high throughput (can send and receive large amounts of data). Great choice for operational and analytical apps including IoT, user analytics and financial data analysis.

Decision Guide: To decide among Bigtable and Firestore, if you are working with data > 1TB of semi structure or structured data which is rapidly changing, and you are working with NoSQL, data is time series or has natural ordering and with big data and running batches or real-time processing or you are running ML algorithms on data, better go with Bigtable.

Making Data Useful and Accessible

Looker

Google Cloud business intelligence platform designed to help teams and individuals analyze, visualize and share data. It includes interactive dashboards, easy to share and understand reports. It helps anyone authorized to ask and answer their questions and creative visualizations. This approach empowers businesses to not only uncover insights but to act on them.

Looker supports BigQuery along with 60+ different SQL DBs. It is 100% web based which makes it easy to integrate and share with multiple teams in organization.

Looker + BigQuery: Rich interactive dashboards and reports without compromising performance, scale, security or data freshness

Streaming Analytics

The process of streaming and analysis of data records continuously instead of batches. Generally it is useful for types of data sources that sends data in small sizes often in KBs in a continuous flow as the data is generated. E.g. click stream, social media feeds, stock market etc.

Google Cloud offers two main streaming analytics to ingest, process and analyze event stream in real time, which makes data more useful and accessible from the instance it is generated.

Pub/Sub

A distributed messaging service that can receive messages from various device streams such as gaming events, IoT devices, and application streams. The name is short for Publisher/Subscriber, or publish messages to subscribers. Ingests hundreds of millions of events per second.

Dataflow

Creates a pipeline to process both streaming (real-time) and batch data. Unifies streaming and batch data analysis and builds cohesive data pipelines.

Process refers to the steps: Extract, Transform and Load data referred to as ETL.

Popular solution for pipeline design is Apache Beam: An open source, unified programming model to define and execute data processing pipelines including ETL, batch and stream processing.

Dataflow allows reliable auto scaling to meet pipeline demands. It is serverless (computing means that a software developer can build and run apps without having to provision or manage backend infrastructure) and fully managed. Google Cloud manages infrastructure tasks on behalf of users like resource provisioning, performance tuning and ensuring pipeline reliability.
